# References Source Of Truth

Generated: 2026-04-25
Sprint: R1.1

## Purpose

This document defines which reference surfaces are canonical, which are generated projections, and which surfaces are supporting judgement or tooling. It prevents later schema, graph, report, retrieval, and diagnostic work from treating every file as equal evidence.

## Layered Canon Rule

| Layer | Canonical role | Edit policy | Evidence weight |
|---|---|---|---|
| `references/external/` | Outside authority: CvTE exams, syllabus, inspection and school standards | refresh or extraction only | strongest factual authority for external claims |
| `references/machine/` | Machine-readable platform registries for units and terms | CLI only | canonical for IDs, dependencies, metadata, and machine use |
| `references/authored/` | Human-authored expert judgement and legacy reference material | human-reviewed edit | useful judgement layer; weaker than external authority for factual alignment |
| `references/owned/` | Owned curriculum-design sources, including the course blueprint | versioned human-reviewed edit | strong source for course intent and target-exercise alignment; not external authority and not a machine registry |
| `references/qc-prompts/` | Quality-control protocols and audit prompts | human-reviewed edit | procedure layer for review work |
| `references/data/` | Governance data: inventories, sprint metadata, future audits and evidence data | generated or sprint lifecycle | canonical only for its declared governance artifact |
| `reports/` | Generated projections and diagnostic views | generated by script | never primary evidence; useful for surfacing issues |
| `build-scripts/references/` | Reference mutation, extraction, validation, and inventory tooling | code review | tooling, not reference evidence |
| `build-scripts/reports/` | Report and dashboard generators | code review | tooling, not reference evidence |
| `references/reference-team-roadmap.md` | Team planning and sprint state | human-reviewed edit | planning authority, not curriculum evidence |

## Canonical Data Rules

1. External sources are canonical for what outside authorities published.
2. Machine registries are canonical for platform IDs, unit relationships, terms, and machine-readable metadata.
3. Generated reports are projections and must be reproducible from source data or tooling.
4. Owned curriculum-design sources define course intent, paragraph sequence, and target-exercise alignment. Owned prose alone cannot mint or mutate machine units.
5. Authored references can guide interpretation, but factual curriculum/exam claims need external or exercise evidence.
6. Sprint metadata records process state. It does not make a pedagogical claim true.
7. Future JSON reports should become the machine-readable report layer; Markdown should become a human projection.

## Protected Surfaces

These surfaces must not be hand-edited:

- `references/machine/`
- `references/external/`

Machine-reference changes flow through CLI scripts. External-reference changes flow through refresh or extraction workflows.
