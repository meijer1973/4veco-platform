# references/machine/

Machine-edited references. Files in this folder are **never hand-edited**. All changes flow through CLI scripts under `build-scripts/references/` that validate, write, and regenerate JSON companions atomically.

## Policy

- **No hand-edits.** If you need to change something here, use the appropriate CLI command (or, later, the `/unit` skill that wraps the CLI). Hand-edits do not survive — the next script run or the next build will revert or flag them.
- Every markdown file here has a generated JSON companion consumed by code.
- Every change is validated at write time: schema, cross-refs, DAG integrity, terminology coherence.
- Integrity reports (under `reports/`) regenerate every build and fail CI on structural violations.

## Initial contents

| File | Description | Edit via |
|---|---|---|
| `micro-teaching-units.md` | Canonical registry of every teaching chunk (math, concept, procedure, reasoning) across the platform | `build-scripts/references/unit-*.js` |
| `micro-teaching-units.json` | Machine projection of the above | generated by `build-scripts/references/build-unit-index.js` |

## Rationale

A catalog of 40–80+ teaching units with DAG dependencies, exam-code cross-references, and procedure consistency cannot be maintained by hand without silent drift. Machine-only editing keeps the catalog internally consistent regardless of how many people or agents touch the platform, and how fast.

See `knowledge/micro-teaching-units-plan.md` for the full architecture.
