# Sprint R7.5: Merge Readiness and Main Sync

## Goal

Sync `refsys/R0.2-reference-baseline-inventory` with `origin/main`, resolve merge-readiness risks, rerun the required validators, and produce a merge-readiness report for lead-developer review.

## Context

R7.4 is closed as `pass_with_conditions`, but the branch is diverged from `main`. The lead developer authorized only this merge-readiness sprint before any merge decision or new roadmap work.

- `docs/sprints/R7.5-plan.md`
- `references/data/sprints/R7.5.plan.json`
- `references/data/sprints/R7.5.result.json`
- `reports/sprints/R7.5-baseline.md`
- `reports/sprints/R7.5-result.md`

- roadmap artifact relocation under `docs/roadmaps/`
- generated reports under `reports/`
- derived RAG data under `references/data/rag/`
- sprint/roadmap status files

## Forbidden paths

- hand edits to `references/machine/`
- hand edits to `references/external/`
- student diagnostics
- adaptive routing
- student-facing AI
- automatic lesson sequencing
- automatic mastery or summative decisions
- RAG follow-up implementation (`RAG-01` through `RAG-04`)
- R8, R9, R10, course-quality, or student-facing work

## Inputs

- `origin/main`
- current branch `refsys/R0.2-reference-baseline-inventory`
- R7.4 closure artifacts
- R3.2 mutation logs
- current validators under `build-scripts/`

## Outputs

- Main-sync merge commit if needed.
- Duplicate R5 gate namespace normalization.
- Stray handoff roadmap artifact moved or classified.
- Merge-readiness JSON and Markdown report.
- R7.5 sprint result logs.

## Operationalized sprint procedure

1. Establish baseline: current branch, clean/dirty status, branch divergence, R5 duplicate gate paths, stray roadmap artifact, mutation logs, and validator availability.
2. Fetch `origin` and merge `origin/main` into the current branch. Stop if conflicts cannot be resolved within the authorized merge-readiness scope.
3. Resolve merge conflicts conservatively, preserving R7.4 closure and main-line changes where both are valid.
4. Normalize duplicate R5 gate paths by making `reports/review-gates/GATE-R5-alignment-graph/` canonical and marking `GATE-R5.3-alignment-graph/` deprecated or removing it if safely redundant.
5. Move or classify `references/final-codex-handoff-roadmap original.md` outside the reference corpus unless there is a deliberate source-corpus reason to keep it.
6. Verify protected machine changes are backed by R3.2 mutation logs and run reference/schema/report/RAG/review-gate validators.
7. For existing gate artifacts, verify that the prior human-gate records still contain calibration questions, answer recording, pattern analysis, targeted follow-ups, closure proposal, and explicit human confirmation after the main sync.
8. Regenerate affected reports and derived RAG data only through scripts.
9. Write the merge-readiness report with required yes/no fields, test results, conflict status, and ready-for-merge decision.
10. Run sprint bundle/result checks and stop before merge-to-main or any new phase.
