# Sprint R7.1: Deterministic Chunk Layer

## Goal

Build the first internal deterministic retrieval chunk layer. This is not student-facing.

## Context

R5.3 allows internal retrieval development/evaluation only if authority, edge status, diagnostic flags, and generated-report warnings are preserved. R7.1 creates the source-ranked chunk index used by R7.2 and R7.3.

## Operationalized sprint procedure

1. Establish baseline: no deterministic RAG chunk index exists.
2. Build chunks for allowed source types: external source, machine unit, machine term, exam question, target exercise, authored reference, quality report, alignment edge, and evidence anchor.
3. Include source path, source type, authority level, entity IDs, evidence IDs, edge statuses, public-citation flag, curriculum authority, text, generator, timestamp, source hash, and generated-report safeguards.
4. Sort chunks deterministically by chunk ID.
5. Validate required fields, valid enums, unique IDs, and generated-report restrictions.
6. Write result artifacts and roadmap completion state.
7. Stop if chunks require hand-editing protected references or flatten generated reports into primary evidence.
