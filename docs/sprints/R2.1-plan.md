# Sprint R2.1: Full Empty-Needs Audit

## Goal

Run a full non-mutating audit over all live units with empty `needs` and produce review-ready JSON and Markdown outputs.

## Context

R1.3 added schema support for zero-needs and prior-knowledge review. R2.1 uses that schema direction to classify current empty-needs units as machine suggestions only. Dependency changes are not allowed in this sprint.

## Operationalized sprint procedure

1. Read the prior-knowledge schema, machine unit catalog, and R2 roadmap rules to define a non-mutating audit of live units with empty `needs`.
2. Build the audit script so it produces machine suggestions only, with categories that are review-ready but not authoritative.
3. Write JSON and Markdown audit outputs that identify evidence, uncertainty, and proposed review direction without changing dependencies.
4. Add a validator that checks audit shape and consistency against the schema.
5. Run the sprint plan checker, audit builder, audit validator, inventory validators, dashboard builder, and sprint result checker.
6. Stop before editing `references/machine/` or accepting recommendations; human review is required before any dependency correction can become authoritative.
