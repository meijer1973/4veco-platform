## Operationalized sprint procedure

1. Read `review-packet.md`, all three subagent review JSON files, and the R2.3 baseline.
2. Ask 8-10 calibration questions grounded in the review packet. Cover strong candidates, suspicious candidates, underbouw assumptions, ambiguous units, and evidence sufficiency.
3. Record each human answer in `human-interview.md`.
4. Analyze answer patterns and choose one next mode:
   - `continue_targeted`
   - `batch_confirm_low_risk`
   - `pause_for_missing_evidence`
   - `rerun_audit_with_revised_rubric`
   - `schema_change_required`
   - `propose_gate_closure`
5. Ask targeted follow-ups or batch-confirm low-risk rules based on the selected mode.
6. Draft a closure proposal that names the gate status, accepted classes of recommendations, held/rejected classes, allowed next sprint, blocked next sprint, and conditions.
7. Ask for explicit human confirmation of the closure proposal.
8. Only after explicit confirmation, write `human-interview.md`, `gate-closure.json`, R2.3 result files, and run the gate validator plus sprint acceptance tests.
9. Stop if the human reviewer does not confirm a closure proposal. Do not apply dependency corrections in R2.3.
