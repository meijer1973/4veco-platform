## Operationalized sprint procedure

1. Establish the baseline: count active lesson files and paragraph directories, count stale target-exercise blueprint references, and confirm protected `references/machine/` and `references/external/` are not in scope.
2. Build an owned-source registry generator that scans the canonical blueprint, blueprint metadata, target-exercise index, and active Book 1 lesson corpus. The generator must classify owned surfaces as authored source, generated projection, exercise evidence, answer model, planning artifact, or implementation output.
3. Encode authority weights and use policy in the registry. Owned source material may support course design, RAG retrieval, target-exercise alignment, and projection edges; it may not override external authority or reviewed machine registries.
4. Add a registry schema and validator. The validator must fail if the registry lacks the canonical blueprint, lacks active lesson-source classes, collapses projection edges into evidence edges, or treats generated lesson output as primary authority.
5. Repair stale target-exercise blueprint references from `knowledge/course_blueprint_v4.md` to `references/owned/course-blueprint-v4.md` through a script, then validate that no stale references remain in the target-exercise index.
6. Regenerate the reference inventory so `source_manifest.json` and `document_inventory.json` include the new registry/report/script surfaces.
7. Produce the CP-2 owned-source scope review packet. It must include calibration questions, answer recording fields, pattern analysis criteria, targeted follow-ups, a closure proposal section, and explicit human confirmation fields before any gate closure is recorded.
8. Run validators. If validation finds a protected-surface edit, missing authority classification, projection/evidence collapse, or stale blueprint reference that cannot be repaired safely within this sprint, stop and report instead of patching around it.
9. After CP-2 human review is available, record the result, update the roadmap, and close R9.1 only if the gate decision is `pass` or `pass_with_conditions`.
