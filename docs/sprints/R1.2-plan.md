# Sprint R1.2: Core JSON Schemas

## Goal

Create the first core JSON schema contract for reference entities before later sprints build evidence anchors, alignment graphs, reports, retrieval chunks, diagnostics, games, and simulations.

## Context

R1.1 defined source-of-truth layers. R1.2 turns that policy into schema files under `references/schemas/`.

## Operationalized sprint procedure

1. Read the source-of-truth policy and roadmap schema phase to identify the first schema surfaces needed before evidence, graph, reports, retrieval, feedback, games, and simulations.
2. Create or update only schema files and schema validators; keep this sprint about structural contracts, not data correction.
3. Ensure schemas preserve layer boundaries from `references/SOURCE_OF_TRUTH.md` and do not make generated reports more authoritative than source references.
4. Regenerate inventory and dashboard projections so the new schema layer is tracked.
5. Run the sprint plan checker, core schema validator, inventory validators, dashboard builder, and sprint result checker.
6. Stop if a schema decision would force machine-reference data changes or student-routing authority; that requires a later sprint and the relevant human review gate.
