# Sprint R9.2: Content Graph Projection

## Goal

Project owned course and lesson content into a governed content graph for navigation, coverage, and internal retrieval without turning owned exposition or generated outputs into curriculum authority.

## Context

R9.1 closed CP-2 as `pass_with_conditions`. R9.2 is authorized only if owned-source edges default to projection edges and evidence edges are reserved for explicitly evidence-qualified records.

R9.2 uses:

- `references/data/owned-source-registry.json` as the source-surface policy input.
- `references/owned/course-blueprint-v4.md` as the owned course-design backbone.
- `references/authored/course-target-exercises.json` as owned exercise evidence.

The existing R5 alignment graph remains governed by its own gate. R9.2 creates a separate owned-content projection graph so RAG can retrieve owned content without weakening R5 alignment authority.

## Operationalized sprint procedure

1. Establish the baseline: confirm no owned-content graph exists, read CP-2 conditions, and record target-exercise and lesson-corpus counts.
2. Build the owned-content graph from target exercises and active lesson file discovery. Use projection edges by default. Only target-exercise and owned exercise markdown may create `owned_exercise_evidence` edges, and those edges must not be external authority.
3. Include edge-level metadata for edge type, source path, source surface type, source status, authority level, authority weight, curriculum authority, primary evidence, generated-artifact warning, evidence status, related units, and related terms.
4. Link blueprint chapters/paragraphs, target exercises, paragraph markdown, opgaven markdown, answer models, visual sources, summaries, and generated outputs where a paragraph ID is available.
5. Generate coverage reports that summarize paragraph, unit, term, source-surface, projection-edge, evidence-edge, and generated-artifact counts.
6. Extend RAG chunk generation to include owned-content projection edges using a non-authoritative chunk shape. RAG retrieval must preserve edge status and generated-artifact warnings.
7. Run validators. Stop if generated outputs are marked primary evidence, if default owned-source edges are not projection edges, if evidence edges are not explicitly evidence-qualified, or if protected reference data would need hand edits.
8. Close R9.2 with result artifacts and roadmap updates only after validators and retrieval smoke tests pass.
