# Sprint R6.2: Reference Health Dashboard Data

## Goal

Produce one generated reference-health state file for dashboards and planning.

## Context

R6.1 created JSON-first health reports. R6.2 aggregates them into `reference-health.json` and a readable Markdown projection while preserving R5.3 graph-authority constraints.

- `references/data/sprints/R6.2.result.json`
- `reports/sprints/R6.2-baseline.md`

## Forbidden paths

## Outputs

- `reports/json/reference-health.json`
- `reports/markdown/reference-health.md`

## Operationalized sprint procedure

1. Establish baseline from R6.1 JSON reports and R5.3 gate closure.
2. Generate a single reference-health JSON file containing catalog counts, coverage summaries, QC issues, schema/report status, graph status, gate status, allowed downstream uses, and blocked downstream uses.
3. Include explicit `graph_authority` policy fields from R5.3.
4. Generate a readable Markdown projection.
5. Validate required sections and policy booleans.
6. Write result artifacts and roadmap completion state.
7. Stop if graph authority constraints cannot be represented.
