# Sprint R4.5: Apply Reviewed Micro-Unit Corrections

## Goal

Apply only the human-reviewed R4.4 micro-teaching-unit corrections through the reference CLI, then regenerate and validate the affected reference, report, and RAG surfaces.

## Context

R4.4 produced a non-mutating packet for remaining micro-unit quality issues. Human review approved a bounded R4.5 mutation set: two canonical B-domain terms, a small deterministic dependency-edge set, and high-priority Book 1/2 foundation units from the owned blueprint. Ambiguous unit-design cases remain held for later review.

## Operationalized sprint procedure

1. Record the human R4.5 authorization as a decision artifact.
2. Encode the approved term additions, dependency edges, unit additions, true roots, and held design-review items in a machine-readable mutation spec.
3. Repair the term CLI domain list only enough to support approved B-domain term additions.
4. Apply the mutation spec through CLI commands only:
   - add `schaarste` and `alternatieve-kosten`;
   - update B02 to the canonical term slug;
   - dry-run all approved dependency edges before applying them;
   - apply only approved dependency edges;
   - mint only approved new micro-units.
5. Stop if any CLI mutation fails validation, creates a cycle, creates an unresolved ID, or requires hand-editing a protected machine file.
6. Regenerate affected reports, reference health, RAG chunks, retrieval evals, and inventories.
7. Validate schemas, reports, sprint bundle, and generated outputs.
8. Update the roadmap ledger and current catalog state.
