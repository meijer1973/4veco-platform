# Sprint R9.01: Legacy Folder Rename

## Goal

Rename repo-local `module-3` folders to legacy-neutral folder names, while preserving frozen external target behavior and legacy maintenance guardrails.

## Context

R9.0 removed Module 3 references from the reference corpus, but two repo-local folders still used `module-3` as their folder name: `source-data/module-3/` and `build-scripts/content/module-3/`. The user asked to fix the folder naming. The frozen external lesson target may still contain its historical name because changing it could affect legacy deployment and student-localStorage integrity.

## Allowed paths

- `docs/sprints/R9.01-plan.md`
- `references/data/sprints/R9.01.plan.json`
- `references/data/sprints/R9.01.result.json`
- `reports/sprints/R9.01-baseline.md`
- `reports/sprints/R9.01-result.md`
- `reports/sprints/R9.01-diff-summary.md`
- `source-data/legacy-target/`
- `build-scripts/content/legacy-target/`
- scripts, README files, and roadmap/sprint records that reference the renamed repo-local folders
- generated inventories after folder rename

## Forbidden paths

- hand edits to `references/machine/`
- hand edits to `references/external/`
- renaming the external frozen lesson target directory
- changing legacy deploy behavior beyond repo-local path references
- changing student localStorage keys or runtime storage behavior
- student diagnostics, adaptive routing, student-facing AI, automatic sequencing, mastery, or summative decisions

## Inputs

- current repo-local directories `source-data/module-3/` and `build-scripts/content/module-3/`
- current references in `scripts/deploy.js`
- current references in `build-scripts/references/rewire-skilltree.js`
- repo documentation and sprint artifacts that mention the repo-local folders

## Outputs

- `source-data/legacy-target/`
- `build-scripts/content/legacy-target/`
- updated code/doc references to the renamed repo-local folders
- regenerated source manifest and document inventory
- R9.01 result and diff summary

## Operationalized sprint procedure

1. Establish baseline by listing repo-local directories whose names match `module-3`, and distinguish them from the external frozen lesson target name.
2. Use git-aware moves for the two repo-local folders only. Stop if either target path already exists or if the move would touch the external lesson target.
3. Update scripts and documentation that reference repo-local paths so validators and legacy helper scripts resolve `legacy-target` instead of `module-3`.
4. Run targeted searches for old repo-local path strings, regenerate inventories, and run sprint/reference validators.
5. Record whether any historical or external-target references remain. Those may remain if they are not repo-local folder names and are needed for guardrail accuracy.

## Rollback plan

If the rename breaks validation, revert the R9.01 commit. The external frozen lesson target is not renamed by this sprint.

No human gate is required. Report back with the renamed folders and any remaining Module 3 mentions that refer to the external frozen target or historical content.
