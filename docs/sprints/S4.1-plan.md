# Sprint S4.1: Exercise Overlay Conditions And Calibration

## Goal

Implement the CP-3 closure conditions in the exercise overlay layer before any bulk metadata extension: source-annex gap tracking, provisional operation wording, scaffolding calibration notes, graph-spec refinement planning, and stronger product-boundary warnings.

## Context

`GATE-CP3-schema-extension-dry-run` closed as `pass_with_conditions`. It approved the S4 dry-run overlay shape only with conditions.

S4.1 is a hardening sprint, not a source-mutation sprint and not a bulk metadata backfill sprint.

CP-3 conditions carried forward:

- track Tier A source annex extraction gaps before broad Tier A overlay backfill
- mark `exercise_operations` as provisional until a governed operation registry exists
- add reviewer calibration notes for scaffolding scale values
- refine `graph_spec` representation values during broader coverage work
- strengthen warnings around `instructional_role: diagnostic`
- preserve product-boundary flags: no student diagnostics, adaptive routing, student-facing AI, mastery decisions, summative use, or automatic sequencing

## Operationalized sprint procedure

1. Record S4.1 plan and baseline before editing overlay artifacts. Confirm that CP-3 is closed as `pass_with_conditions` and that protected source data starts unchanged.
2. Add condition artifacts under `references/data/exercises/`: source-annex gap tracking, scaffolding calibration notes, graph-spec representation refinement planning, and product-boundary warning language.
3. Update the overlay schema and existing dry-run overlays so `exercise_operations` are explicitly provisional, source-annex gaps are tracked, and product-boundary warnings are non-optional. Stop if this would require source mutation or bulk backfill.
4. Implement a read-only CP-3 condition validator. It must check the condition artifacts, confirm unresolved source-value gaps are tracked with proof-to-close, and ensure product-boundary authorizations remain false.
5. Generate a concise condition report under `reports/json/` and `reports/markdown/`.
6. Regenerate source manifest and document inventory, update roadmap versioning, and run acceptance checks. Do not close or start Sprint 12-style bulk backfill from this sprint.

## Rollback plan

If the condition hardening is wrong, revert the S4.1 commit to remove the condition artifacts, schema/validator updates, reports, and roadmap bookkeeping. No rollback is needed for `references/machine/` or `references/external/` because S4.1 must not edit them.

## Human review required

No new human gate is required if all validators pass and S4.1 stays inside the CP-3 allowed scope. Human review is required before any source mutation, broad Tier A overlay backfill, or product use expansion.
