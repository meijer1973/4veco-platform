# Sprint R1.1: Source-of-Truth Decision

## Goal

Record the canonical source-of-truth policy for reference files so later schema, graph, report, and retrieval work does not confuse external authority, machine registries, authored judgement, generated reports, and tooling.

## Context

R0.2 created `references/data/source_manifest.json` and `references/data/document_inventory.json`. R1.1 converts that inventory into a human-readable policy and a validator.

## Operationalized sprint procedure

1. Read the R0.2 inventory outputs and the roadmap source-of-truth phase to identify every reference layer that needs an authority and edit-policy rule.
2. Write `references/SOURCE_OF_TRUTH.md` so external authority, machine registries, authored judgement, generated reports, dashboards, and tooling each have clear status.
3. Add a validator that checks the policy file names the required source layers and protected surfaces.
4. Regenerate inventory and dashboard projections so the policy change is visible to later sprints.
5. Run the sprint plan checker, source-of-truth validator, inventory validators, dashboard builder, and sprint result checker.
6. Stop if the policy implies a machine-reference content mutation; this sprint records governance only and any pedagogical decision must remain for human review in a later gate.

## Rollback plan

Delete the R1.1 sprint files, `references/SOURCE_OF_TRUTH.md`, and `build-scripts/references/check-source-of-truth.js`. Regenerate the inventory and dashboard from the previous roadmap state.

## Human review required

Human review is useful before later sprints rely on this policy as a formal governance rule, but R1.1 itself does not require a pedagogical decision.
