# Sprint R6.1: JSON-First Reports

## Goal

Create JSON-first health and coverage reports so dashboards and later retrieval tooling consume structured report data rather than scraping Markdown.

## Context

R5.3 closed the alignment graph gate as `pass_with_conditions`. R6.1 is allowed to generate structured internal reports, but generated reports remain diagnostic and must not become primary evidence.

## Operationalized sprint procedure

1. Establish baseline from current reports and catalog counts.
2. Implement `generate-all.js` to emit a normalized JSON contract for every required health report and a readable Markdown projection.
3. Implement `validate-report-json.js` to enforce metadata, status, source files, and issue fields.
4. Generate all reports.
5. Validate all generated JSON reports.
6. Write result artifacts and roadmap completion state.
7. Stop if report JSON cannot preserve issue severity, evidence path, next action, and proof-to-close.
