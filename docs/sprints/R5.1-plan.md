# Sprint R5.1: Evidence-Anchor Layer

## Goal

Create the first governed evidence-anchor layer: schema, source-ranking policy, initial high-risk anchor records, claim links, and a validator.

## Context

R4 completed cleanup queues for term slugs, exam-question gaps, and blueprint flags. R5 begins evidence governance. The roadmap requires exact proof anchors for important claims, prerequisite edges, term definitions, and unit decisions before alignment graph work begins.

This sprint must create a minimal usable evidence layer without turning generated reports into primary evidence.

## Operationalized sprint procedure

1. Establish baseline: existing schemas are present, but no `references/data/evidence-anchors.json` and no validator exist.
2. Extend the evidence-anchor and claim schemas so anchors carry source layer, authority rank, locator, excerpt/summary, extraction method, and confidence.
3. Create a small source-ranking policy in `references/data/evidence-anchors.json` that orders external authority above authored source material, machine registry, human gate decisions, and generated reports.
4. Add initial anchors and claims for high-risk areas named by the roadmap: elasticity prerequisite decisions, labor-market sequencing, and exam-question gap claims. Use generated reports only as secondary/supporting evidence, never as primary proof.
5. Build `check-evidence-anchors.js` to verify schema-version fields, unique IDs, source paths, allowed authority ranks, valid source layers, locator objects, claim-to-anchor links, and no missing evidence IDs.
6. Generate Markdown and JSON status reports from the validator.
7. Write result artifacts and update the roadmap. Stop before constructing alignment graph edges; R5.2 owns that.
