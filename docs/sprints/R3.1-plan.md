# Sprint R3.1: Reference CLI And Documentation Completion

## Goal

Audit and tighten the reference CLI layer so reviewed R2.4 corrections can later be applied without hand-editing protected machine references.

## Context

R2.4 completed a non-mutating evidence and unit-design packet. R3.2 remains blocked until the R2.4 packet is human-reviewed for mutation decisions and the CLI/documentation/validator path can express accepted corrections safely.

R3.1 is non-mutating with respect to protected reference data. It may update CLI code, validators, documentation, and coverage reports.

## Operationalized sprint procedure

1. Establish the baseline from R2.4 and confirm `references/machine/` and `references/external/` are not edited.
2. Audit the R2.4 packet mutation needs against actual CLI commands: dependency edges, zero-needs status, assumed-prior-knowledge classification, D04 lifecycle changes, term-link updates, and evidence-anchor storage.
3. Inspect the unit parser, formatter, JSON projection, and validators to see whether zero-needs review fields survive a CLI write.
4. Implement only CLI-readiness fixes that do not mutate protected reference data. Stop before applying any R2.4 correction.
5. Update `build-scripts/references/README.md` so it no longer claims the CLI suite is future-only.
6. Write `reports/json/reference-cli-coverage.json` and `reports/reference-cli-coverage.md` with per-mutation readiness and remaining gaps.
7. Add a deterministic checker for the coverage report and relevant scripts.
8. Run validators and stop if a required R2.4 mutation type still has no CLI path and is not explicitly documented as a blocker.
9. Update the roadmap: R3.1 complete, R3.2 still blocked pending human mutation review of R2.4.
