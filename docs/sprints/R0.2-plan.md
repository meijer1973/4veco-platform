# Sprint R0.2: Reference Baseline Inventory

## Goal

Create a complete, machine-checkable inventory of the current reference system before schema, graph, evidence-anchor, or retrieval work changes the shape of the data.

## Context

The references roadmap now treats the reference folder as the governed educational knowledge base. Before later phases change schemas or data, the team needs a baseline inventory that records what files exist, what layer they belong to, who owns them, how they may be edited, and how they refresh.

This sprint implements only the inventory and its validation. It does not change machine reference content or external source mirrors.

## Operationalized sprint procedure

1. Read the roadmap, sprint scaffold docs, existing reference folders, reports, and reference build scripts to define the baseline surface before changing schemas or data.
2. Record the baseline state in the R0.2 sprint files, then build a manifest and document inventory that classify source layer, ownership, edit policy, refresh path, and generated status.
3. Add validators that prove the manifest and document inventory are internally consistent and that protected reference surfaces remain unchanged.
4. Regenerate dashboard and inventory projections only from the declared inputs.
5. Run the sprint plan checker, inventory builders, inventory validators, dashboard builder, and sprint result checker.
6. Stop if any protected `references/machine/` or `references/external/` content would need hand editing; that requires a later CLI or refresh sprint and human review if it changes educational authority.

## Rollback plan

Delete the R0.2 sprint files, inventory JSON files, and R0.2 inventory scripts. Restore `references/reference-team-roadmap.md` and the internal dashboard generated files from the previous commit if the adopted roadmap/inventory direction is rejected.

Protected reference registries under `references/machine/` and external mirrors under `references/external/` are not modified by this sprint.

## Human review required

Human review is required before later sprints rely on the inventory as a source-of-truth map. No pedagogical decisions are made in this sprint.
