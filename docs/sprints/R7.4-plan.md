## Operationalized sprint procedure

1. Establish baseline: R7.3 evals pass and no R7 gate packet exists yet.
2. Prepare calibration questions for the human gate so review can test retrieval behavior, authority ranking, generated-report handling, pending-review handling, and allowed/blocked downstream uses.
3. Prepare answer recording surfaces by naming the review packet and the gate artifacts where future human answers must be logged.
4. Record packet evidence from R7.1 through R7.3, including sample queries, evaluation results, known failures, evidence-anchor behavior, and proposed allowed/blocked uses.
5. Analyze pattern risks across the three review lenses: retrieval behavior, authority ranking, and evidence surfacing.
6. Add targeted follow-ups where the packet reveals limits, especially lexical ranking weakness, large target-exercise chunks, and generated-report diagnostics.
7. Include a closure proposal with valid gate statuses, but do not create final closure. Explicit human confirmation is required before any R7.4 closure record is created.
8. Run the packet validator and stop if it finds missing review sections, missing blocked-use policy, missing warning behavior, or a closed gate-closure file.
