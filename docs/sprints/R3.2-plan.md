# Sprint R3.2: Apply Reviewed Empty-Needs Corrections

## Goal

Apply only the R2.4 mutation-review approved dependency edges and foundational A-domain zero-needs classifications through the reference CLI.

## Context

R2.4 produced a non-mutating evidence packet. R3.1 confirmed the CLI layer is ready with blockers. The human mutation-review decision now authorizes a bounded R3.2 mutation set.

## Operationalized sprint procedure

1. Record the human mutation-review decision as a gate artifact.
2. Confirm the approved mutation set exactly matches the decision JSON.
3. Add CLI dry-run support if missing for `unit-add-dep` or `unit-update`; do not mutate references before dry-run support exists.
4. Run dry-run commands for every approved edge and zero-needs status update.
5. If any dry-run fails, stop and write a failure result.
6. Apply approved edges through `unit-add-dep.js`.
7. Apply approved zero-needs classifications through `unit-update.js`.
8. Run full reference validation.
9. Regenerate affected reports.
10. Write the mutation log, result, diff summary, and result JSON.
11. Update the roadmap: R3.2 complete, D04 and held items routed to later review.
