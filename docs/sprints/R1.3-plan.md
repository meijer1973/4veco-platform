# Sprint R1.3: Unit Prior-Knowledge Schema Extension

## Goal

Add schema support for empty-`needs` governance so later audits can distinguish true zero-prerequisite units, assumed lower-school knowledge, false-zero hidden prerequisites, and ambiguous cases.

## Context

R1.2 created the core unit schema. R1.3 extends it with prior-knowledge review fields and adds a review-entry schema for non-mutating audits.

## Operationalized sprint procedure

1. Read the unit schema, source-of-truth policy, and R2 empty-needs roadmap rules to define the minimal schema support needed for later non-mutating review.
2. Extend the unit schema and add a prior-knowledge review-entry schema that can distinguish true zero-prerequisite units, assumed prior knowledge, false-zero gaps, and ambiguous cases.
3. Add a validator that proves the new schema contract can be loaded and checked without editing the machine unit catalog.
4. Regenerate inventory and dashboard projections so the schema extension is visible to later sprints.
5. Run the sprint plan checker, core schema validator, prior-knowledge schema validator, inventory validators, dashboard builder, and sprint result checker.
6. Stop before accepting any dependency correction; this sprint enables later review and any pedagogical decision waits for the R2 human review gate.
