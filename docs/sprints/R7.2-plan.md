# Sprint R7.2: Hybrid Retrieval

## Goal

Implement an internal retrieval prototype over the deterministic chunk layer and alignment graph.

## Context

R5.3 allows internal retrieval development/evaluation only when retrieval preserves edge status, source authority, evidence anchors, diagnostic-only flags, pending-review flags, and generated-report warnings.

## Allowed paths

- `docs/sprints/R7.2-plan.md`
- `references/data/sprints/R7.2.plan.json`
- `references/data/sprints/R7.2.result.json`
- `reports/sprints/R7.2-baseline.md`
- `reports/sprints/R7.2-result.md`
- `reports/sprints/R7.2-diff-summary.md`
- `build-scripts/rag/query.js`
- `build-scripts/rag/validate-query-output.js`
- `references/reference-team-roadmap.md`

## Forbidden paths

- `references/machine/`
- `references/external/`
- generated lesson output
- student diagnostics
- adaptive routing
- student-facing AI
- vector search or external AI calls

## Inputs

- `references/data/rag/chunk_index.jsonl`
- `references/data/alignment-graph.json`

## Outputs

- Internal query command and validator.

## Operationalized sprint procedure

1. Establish baseline: chunk index exists and validates; no internal query command exists.
2. Implement lexical/entity/graph search over JSONL chunks.
3. Support required query modes: unit, term, exam code, and quality issue.
4. Preserve source path, source type, authority level, entity IDs, evidence IDs, edge statuses, diagnostic-only flags, pending-review flags, generated-report warnings, and non-primary evidence warnings.
5. Validate JSON output from required query modes.
6. Write result artifacts and roadmap completion state.
7. Stop if retrieval flattens all edges into equal truth or hides generated-report/diagnostic warnings.
