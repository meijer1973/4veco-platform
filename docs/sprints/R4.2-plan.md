# Sprint R4.2: Exam-Question Extraction Gap Closure

## Goal

Create a bounded, evidence-carrying gap report for `references/external/exam-questions.json` so missing required-skill and exam-code annotations become a concrete reviewed patch queue instead of an open-ended audit.

## Context

R4.1 completed the unit-term slug migration. R4.2 now turns the exam-question extraction backlog into production signal. Exam questions are an external-reference extraction surface: this sprint may inspect and report on `references/external/exam-questions.json`, but it must not hand-edit it.

The sprint is intentionally narrow. It closes the current extraction-gap discovery step by producing a deterministic gap report, candidate patch queue, and stop conditions for later reviewed mutation or re-extraction work.

## Operationalized sprint procedure

1. Establish baseline counts for exam questions, exam papers represented, questions without `required_skills`, questions without `exam_codes`, invalid unit IDs, invalid exam codes, and missing question types.
2. Build a non-mutating audit script that reads the exam-question extraction, unit registry, syllabus codes, and existing audit files. The script must only write reports under `reports/`.
3. For each gap, attach concrete evidence from the actual question record: exam code, opgave, question number, page range, question text snippet, current annotations, and any candidate annotations from the audit files.
4. Classify each queue item into a bounded next action: derive exam codes from existing unit IDs, review and apply existing-unit annotations, map or mint unresolved audit slugs through later unit workflow, or manual question review.
5. Validate that no candidate annotation is applied to `references/external/` or `references/machine/` during R4.2. Any mutation must stop for a later reviewed workflow.
6. Write result artifacts and update the roadmap. Acceptance requires the audit script, sprint result check, sprint bundle check, and `git diff --check`.

Revert the R4.2 completion commit. That removes the audit script, generated gap reports, sprint artifacts, and roadmap status update together. No protected reference data should need rollback because this sprint is non-mutating.

## Human review required

No human gate is required to produce the non-mutating gap report. Human review is required before any later workflow applies candidate annotations, mints units, or changes protected reference surfaces.
