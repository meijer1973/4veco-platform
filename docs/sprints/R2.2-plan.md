# Sprint R2.2: Subagent Review For Empty Needs

## Goal

Review the R2.1 empty-needs audit from pedagogy, data-integrity, and evidence perspectives, then prepare a human-review packet for R2.3.

## Context

R2.1 produced machine-suggested classifications. R2.2 does not accept or apply those classifications. It prepares the next human review gate.

## Operationalized sprint procedure

1. Read the R2.1 audit outputs, source-of-truth policy, and R2 review-gate rules before preparing any review packet.
2. Produce separate pedagogy, data-integrity, and evidence review files so the human reviewer can see disagreements and uncertainty by perspective.
3. Build a review packet that summarizes candidate classes, risks, evidence gaps, and questions for the later human gate.
4. Add a validator that checks the review packet and subagent review files exist and are internally consistent.
5. Run the sprint plan checker, review-packet builder, review validator, inventory validators, dashboard builder, and sprint result checker.
6. Stop before closing the gate or applying dependency corrections; R2.2 prepares human review, while R2.3 must run the structured interview and decision process.
