## Deploy pipeline (documented in CLAUDE.md)

Platform holds build tools + engines + shell templates. **No web pages live in the platform.** Pages are generated into module directories by `scripts/deploy.js`:

```
platform/engines/*.{js,css}           →  <module>/shared/*
platform/build-scripts/platform/*.js  →  <module>/<paragraaf>/*.html
```

Test the skilltree in **`3-Module-3-rewire-test/`**. Module 3 itself is frozen until September 2026 per project memory.

Run from the platform root:
```
node scripts/deploy.js "../3-Module-3-rewire-test"
```

**Golden rule** (now in `CLAUDE.md`): if you don't see a change in the browser, the answer is always "you haven't deployed yet."

## Architecture

### HTML scaffold (in each deployed paragraph file)
Produced by `build-scripts/platform/build-skilltree-shells.js` → `generateHTML()`. ~100 lines per file, same shape across all 22 paragraphs. Contains:
- Top breadcrumb strip (`.st-top-strip` with `§X.Y.Z` chip)
- `<section>` per view: `#st-view-tree`, `#st-view-deps`, `#st-view-expl`, `#st-view-exercise` — toggled via `hidden` attribute
- Persistent `#st-info-overlay` (small centred modal, also toggled via `hidden`)
- Stable slots with unique IDs for every piece of dynamic content (layers, step card, result card, etc.)
- Canonical template: `engines/tests/_wiskundevaardigheden-template.html`

### JS layer (`engines/skilltree-ui.js`, 780 lines)
Single IIFE, plain ES5 (`var`, `function`, `.prototype`). Reads globals, builds an `els` map via `document.getElementById`, populates slots, toggles view visibility. No `root.innerHTML = '<section ...>'`, no `document.body.insertAdjacentHTML()`.

### CSS (`engines/skilltree.css`, ~1,600 lines)
Token-driven. Light default + `html[data-theme="dark"]` override block. Layer identity driven by `[data-layer="0..5"]` attribute (not inline styles). Theme-aware dep-graph SVG via CSS classes (`.st-dep-node-rect`, `.st-dep-edge-met`, `.st-dep-node-iconbg`, …).

### Engine (`engines/skilltree-engine.js`)
Refactored by another developer. Named constants (`MAX_STARS`, `GOAL_COMPLETE_STARS`, `PREREQ_STAR_THRESHOLD`, `MAX_ACTIVE_GOALS`, `LEGACY_PARAGRAPH_KEYS`), six private helpers (`_getVisibleSkill`, `_hasStartedSkill`, `_hasCompletedSkill`, `_collectReachableSkillIds`, `_buildTopologicalOrder`, `_collectSkillsOnActiveGoalPaths`), O(1) BFS pattern, deduplicated BFS walks. Public API unchanged.

## Pending verification

The user saw the first working build but reported three layout issues. All three have been fixed and deployed; user should walk through and confirm:

1. **Dep icon on locked cards**: the `<button>` no longer uses HTML `disabled` (which swallowed child clicks). Now `aria-disabled="true"` + `.st-locked` class, gated in the click-delegation handler. Click the sitemap icon on a card that shows "Eerst: X" — the dep view should open.

2. **Dep view theme inheritance**: SVG elements now carry CSS classes (`.st-dep-node-rect`, `.st-dep-node-id`, `.st-dep-edge-met`, …) whose fills resolve to `var(--layer-accent)`, `var(--green)`, `var(--red)`, etc. Dep view should look properly light in light mode, properly dark in dark mode.

3. **Light-mode tile colour**: `--layer-tint` values bumped from pastel to saturated (`#d5eef3` → `#b3e2ea`, etc.), and card background switched from a tint→white gradient to a solid tint with a subtle white top-highlight. Cards should now read as recognisably teal/amber/green per layer, not white-with-a-hint.

Design mockup was dark-only (`knowledge/Screenshot 2026-04-21 084523.png`). Light mode needed a judgement call on tint intensity; user should sanity-check it matches their taste.

## Conventions established this session

- **HTML owns the persistent scaffold.** JS only binds to existing IDs and populates slots. JS never assembles the top-level scaffold.
- **Layer identity via `data-layer="N"` attribute.** Colours come from CSS. No inline `style="background: #..."` on cards.
- **Theme via `html[data-theme]`.** Toggled by the button in `.st-legend`. Persisted to `localStorage['quizMode']` (shared with the quiz game).
- **Views, not modals.** Full-page experiences (tree / exercise / deps / expl) are `<section class="st-view">` with the `hidden` attribute. Only small info popups remain body-level.
- **ES5 UI + UMD engines.** Matches `quiz-ui.js`, `reasoning-ui.js`, etc. No `const`/`let`/arrow/class. No build step.
- **`aria-disabled`, not `disabled`.** HTML `<button disabled>` swallows clicks on all descendants — actively harmful when child icons should remain clickable.
