# Sprint R8.1: Diff Summary

## Summary

R8.1 adds a scoped internal QC issue model and exposes it to JSON reports, reference health, and RAG chunks as diagnostic context.

## Added

- `references/data/qc/reference-quality-issues.json`
- `build-scripts/references/check-quality-issues.js`
- `reports/json/reference-quality-issues.json`
- `reports/markdown/reference-quality-issues.md`
- `docs/roadmaps/outdated/reference-team-roadmap-v2.12-rx2-first-lane-applied.md`
- R8.1 sprint plan, baseline, result, diff summary, and sprint metadata

## Updated

- `quality-issue.schema.json` now defines the R8.1 category set, severity/status/owner fields, evidence refs, proof-to-close, and internal-only authority boundary.
- `generate-all.js` now emits `reference-quality-issues`.
- `validate-report-json.js` now includes the new JSON-first report.
- `generate-reference-health.js` and `check-reference-health.js` now expose and validate `quality_issue_model`.
- `build-chunks.js` now includes report issue text in generated report chunks.
- Roadmap version moved to `v2.13-r8.1-qc-issue-model`.

## Protected surfaces

No protected reference surfaces were changed.

R8.1 did not modify:

- `references/machine/`
- `references/external/`

## Boundary

QC issues remain internal governance data:

- `internal_only: true`
- `curriculum_authority: false`
- `student_facing_exposure: false`
- `primary_evidence: false`

They are suitable for internal dashboard/RAG diagnostic retrieval, not for automatic curriculum decisions.
